## 監査ログ/スナップショット方式でのマスタの履歴管理
### 概要
---
#### [自作構成]スナップショット方式のみでのマスタの履歴管理
※ライブラリの場合、シグナルを使ったロギングになる。

以下のようなライブラリが使用できない場合に検討(深いカスタマイズが必要になる場合)
* 大規模テーブルに対する余分な履歴の取得が行われパフォーマンスが低下する可能性がある
* 大規模テーブルに対して、別プロセスや別途バッチ起動等で履歴取得を取得したい場合
* マルチテナントで対応でDjango固有の機能から逸脱してカスタマイズしている場合

---
★ORMとの相性〇(過去履歴はhistoryに入るのでORMでも確認は容易)★
